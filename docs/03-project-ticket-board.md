# Project Ticket Board

## Example

As you go through the steps below, please refer to my eShop sandbox project [https://github.com/optivem/atdd-accelerator-eshop](https://github.com/optivem/atdd-accelerator-eshop), see: README.

## 1. Create Project Board

You'll be able to use this project board in further steps to create tasks so that you can stay better organized. Even if you're working by yourself, having a Project Board helps you create tickets for yourself.

Instructions:

1. Open your GitHub Project.
2. Click on "Projects" (you'll see it in the top menu: "Code", "Issues"... "Projects")
3. Click on "+ New project"
4. Choose a template (I chose the Kanban, but you can choose Team Planning if you prefer to work with Sprint-like iterations)
5. Project name: Input your System Name
6. Click on "Create project"
7. In the top-right corner of your Project Board, click on the three dots (...), then "Settings".
8. At the bottom, in the Visibility section, set it as Public.

In the System Repository README file, add the following:

- Project Board - link to Project Board
